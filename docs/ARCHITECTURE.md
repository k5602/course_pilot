# Technical Architecture: Course Pilot

## 1. Local-First Persistence (Diesel + SQLite)

The application stores all user data locally with strong relational integrity.

- **Engine**: Diesel ORM with SQLite (via r2d2 connection pool).
- **Schema**: Auto-generated by diesel_cli into `src/schema.rs`.
- **Migrations**: SQL files in `migrations/`.
- **Key Entities**:
  - `Course`: High-level metadata (title, source URL).
  - `Module`: Logical segments created by the BoundaryDetector.
  - `Video`: Source references, sort order, and completion status.
  - `Exam`: AI-generated MCQs and user results.
  - `Note`: User notes per video, indexed via FTS5.
  - `Tag`: User-defined tags for notes and modules.
  - `UserPreferences`: Cognitive load limits, playback rate, theme.

## 2. Intelligence Stack

### A. BoundaryDetector (The Structurer)

- **Location**: `src/domain/services/boundary_detector.rs`
- **Method**: Title-pattern analysis, not ML embedding.
- **Algorithm**: Scans video titles for numbered patterns (`1.1`, `Module 2`, `Chapter 3.1`, `Week 4`).
- **Fallback**: Batch-size grouping (default 5 videos per module) when title signals are weak.

### B. SessionPlanner (The Scheduler)

- **Location**: `src/domain/services/session_planner.rs`
- **Purpose**: Divides video lists into daily study sessions based on cognitive load.
- **Constraint**: Configurable `cognitive_limit` (minutes per day). Respects module boundaries.

### C. Cloud LLM (The Companion/Examiner)

- **Engine**: Gemini 3.1 Flash Lite (upgraded default model) via `genai` crate (BYOK - Bring Your Own Key).
- **Adapters**: `GeminiAdapter` implementing `LLMProvider`, `ModuleTitleGenerator`.
- **Tasks**:
  - Contextual Q&A (Companion mode).
  - MCQ generation (Examiner mode).
  - Video summarization.
  - Module title generation.
- **Context Optimization**: Restructured pipeline that exclusively passes dense, high-fidelity AI-extracted Video Summaries to the AI chat and quiz systems, bypassing raw noisy transcripts to improve response speeds, reduce API costs, and maximize context concentration.

## 3. Ingestion Pipeline

### YouTube Playlists

1. **Fetch**: Extract raw metadata via `rusty-ytdl`.
2. **Normalize**: Strip noise via `Sanitizer` (emojis, clickbait keywords, tags).
3. **Detect Boundaries**: `BoundaryDetector` groups videos into modules.
4. **Persist**: Store structured course in SQLite; update FTS5 search index.

### Local Media

1. **Scan**: Walk configured directories for MP4/MKV/WEBM files.
2. **Match Subtitles**: Find companion `.srt`/`.vtt` files.
3. **Detect Boundaries**: Same `BoundaryDetector` as YouTube.
4. **Persist**: Store in SQLite with local file paths.

## 4. UI Architecture (GTK4 + libadwaita)

### Framework

- **GTK4** with **libadwaita** for adaptive, native-feeling UI.
- **State**: `Rc<RefCell<AppState>>` shared across widgets.
- **Navigation**: `gtk::Stack` with string-named pages, sidebar toggle buttons.
- **Duration Format**: Consistent, dynamic `H:MM:SS` duration rendering across pages.

### Key UI Subsystems

- **Resume Study Dashboard**: Visual homepage layout with gradient hero banner, glassmorphic stat cards, completion levelbars, and interactive progress cards supporting direct study resume shortcuts.
- **Scroll-Down Video Sub-panels**: Positioned below the video player. Consists of Associated Quizzes, clean Video Summary, and Video Transcript reader. Video player enhanced with fullscreen toggle button, double-click gestures, and `F`/`F11` shortcuts.
- **Interactive Quiz System**: University-grade MCQ presentation with selected option highlighting, correct/incorrect status badges, and expandable explanation drawers showing step-by-step refutations.
- **Dynamic Floating Popup Notes Window**: A globally accessible overlay window triggered via `Ctrl+N` with two states: **Type Mode** (markdown text editor with "Insert Video Reference" hotkey) and **Preview Mode** (Pango-compiled markdown and LaTeX rendering for inline and block mathematical equations), including robust Pango safety escaping.

### Page Pattern

Each page is a struct with:
- `new(state, stack)` - builds widget tree, wires signals.
- `widget()` - returns root widget.
- `refresh()` - updates widget properties from state.

### Async Bridge

GTK owns the main thread. Async work spawns via `tokio_bridge::spawn()`. Results return through `mpsc::channel` polled by `glib::idle_add_local`.

## 5. Hexagonal Architecture

### Domain Layer (`src/domain/`)

Zero external dependencies. Pure business logic.

- **Entities**: Course, Module, Video, Note, Exam, Tag, etc.
- **Ports**: Traits (`Repository`, `LLM`, `Transcript`, `Presence`, `SecretStore`, `YouTube`, `LocalMedia`).
- **Services**: BoundaryDetector, SessionPlanner, Sanitizer, SubtitleCleaner.

### Application Layer (`src/application/`)

Orchestration. Use cases pull ports from `AppContext` and execute business logic.

- **AppContext**: DI container holding all wired dependencies.
- **ServiceFactory**: Creates use cases with correct concrete types.

### Infrastructure Layer (`src/infrastructure/`)

Adapters implementing domain ports.

- **Persistence**: SQLite repositories + FTS5 search.
- **LLM**: Gemini adapter (using **Gemini 3.1 Flash Lite** by default) via `genai` crate.
- **Video**: GStreamer player with `playbin` pipeline, enhanced with custom gestures (double-click fullscreen), dedicated fullscreen toggles, and `F`/`F11` key bindings.
- **Discord**: Rich Presence adapter.
- **Transcript**: YouTube subtitle fetching via `yt-transcript-rs`.
- **Keystore**: OS keyring via `keyring` crate.

### UI Layer (`src/ui/`)

GTK4/libadwaita widgets. Pages, layout, dialogs, toast notifications.

## 6. Security & Privacy

- **BYOK**: No central server. Users provide their own Gemini API keys.
- **Key Storage**: API keys stored in OS keyring (Linux/macOS/Windows native).
- **Zero Telemetry**: No tracking, analytics, or cloud-syncing.
- **Local Data**: All courses, notes, and progress stay on the user's machine.
